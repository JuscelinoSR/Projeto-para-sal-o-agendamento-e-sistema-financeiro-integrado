# Roadmap BeautyJSR

O projeto será construído em 3 etapas para sair rápido do papel, validar o produto com usuários reais e evoluir para uma plataforma SaaS completa.

## Etapa 1: MVP de agendamento

Objetivo: criar a primeira versão utilizável para um salão validar agenda, serviços, profissionais e contato pelo WhatsApp.

Funcionalidades do MVP:

- Site institucional do BeautyJSR com apresentação do produto.
- Página ou seção de serviços do salão.
- Cadastro básico de profissionais.
- Fluxo simples de solicitação de agendamento.
- Botão de contato pelo WhatsApp.
- Fluxo interativo para selecionar serviço, profissional e período.
- Estrutura inicial para painel administrativo.
- Design responsivo para celular e desktop.

Fora do MVP:

- Pagamento online.
- Comissões automáticas.
- Estoque completo.
- Campanhas avançadas de CRM.
- Multiempresa completo com cobrança SaaS.

Critério de sucesso:

- Um cliente consegue entender o serviço, escolher uma opção de atendimento e iniciar contato/agendamento em poucos cliques.

## Etapa 2: Operação do salão

Objetivo: transformar o MVP em uma ferramenta de operação diária.

Funcionalidades principais:

- Agenda com horários, status e profissionais.
- Cadastro de clientes.
- Histórico de atendimentos.
- Serviços com duração e preço.
- Painel administrativo com indicadores básicos.
- Confirmações e lembretes pelo WhatsApp.
- Controle financeiro inicial com receitas e despesas.

Critério de sucesso:

- O salão consegue usar o sistema para organizar a rotina de agenda, clientes e caixa.

## Etapa 3: Plataforma SaaS BeautyJSR

Objetivo: evoluir o produto para uma plataforma multiempresa, white label e escalável.

Funcionalidades principais:

- Multiempresa com isolamento por organização.
- Planos e assinaturas SaaS.
- White label por cliente.
- CRM com automações D+3, D+7, D+14 e D+30.
- Fidelidade, estoque, comissões e avaliações.
- LGPD, auditoria e Row Level Security.
- Internacionalização para português, inglês e espanhol.

Critério de sucesso:

- Vários negócios de beleza conseguem operar na mesma plataforma com dados isolados, identidade própria e planos comerciais diferentes.

## Primeira opção de MVP

A primeira opção recomendada é o MVP de agendamento com WhatsApp.

Nome sugerido: BeautyJSR Agenda.

Entrega inicial:

- Landing page profissional.
- Lista de serviços.
- Lista de profissionais.
- Chamada para agendamento pelo WhatsApp.
- Prévia visual do painel.
- Estrutura para evoluir depois para Next.js, Supabase e autenticação.

Prioridade atual:

1. Melhorar a landing page atual.
2. Criar a seção de serviços reais.
3. Criar a seção de profissionais.
4. Criar botão de WhatsApp com mensagem pronta.
5. Preparar a próxima migração para app Next.js.
